# Session Log

## 概述

本 skill 用于在 Codex 中维护当前工作目录下的中文 Markdown 日志文件 `session-log.md`。它负责创建或更新日志文件、维护头部概览区、追加结构化任务记录，并优先保证中文内容以 UTF-8 方式稳定写入，避免乱码。

## 适用场景

当用户有以下需求时，应使用本 skill：

- 在当前工作目录创建或维护 `session-log.md`
- 为当前任务会话建立结构化日志
- 仅更新日志头部概览
- 检查当前日志状态
- 关闭当前会话的自动记录模式
- 要求日志内容默认使用中文，并尽量避免编码乱码

## Codex 触发规则

在 Codex 中，**不要依赖** `/session-log` 这类 slash command 形式触发本 skill，因为 `/...` 可能会被客户端内置命令系统优先解析。

在 Codex 中，应优先将以下写法视为显式调用：

- `$session-log 初始化日志`
- `$session-log 更新概览`
- `$session-log 查看状态`
- `$session-log 关闭记录`

也可以接受自然语言触发，例如：

- `使用 session-log skill 初始化当前目录日志`
- `使用 session-log skill 更新概览`
- `使用 session-log skill 查看当前日志状态`

说明：

- `$session-log` 后面的内容属于**动作说明**
- 它不是严格的系统级子命令解析器
- 应根据用户语义理解其动作意图

## 动作语义

### 1. 初始化日志

含义：

- 若当前工作目录下不存在 `session-log.md`，则创建它
- 若文件已存在，则继续复用，不覆盖历史内容
- 若概览区缺失，则补齐标准结构
- 开启当前会话的记录模式

开启记录模式后：

- 对后续**任务相关、能够推动问题解决**的有效对话，可自动追加简洁结构化正文记录
- 不记录纯闲聊、寒暄、单句确认或无推进价值的内容

### 2. 更新概览

含义：

- 只更新 `session-log.md` 顶部概览区
- 不要求单独追加一条正文，除非用户明确要求
- 概览应反映**截至当前的整体进度**
- 概览不等于最后一轮对话全文

### 3. 查看状态

含义：

- 返回当前日志文件路径
- 返回文件是否存在
- 返回概览是否存在
- 返回累计记录数、最近更新时间、当前状态、当前阶段
- 结合当前会话上下文，判断当前会话的记录模式是否已开启

### 4. 关闭记录

含义：

- 关闭当前会话的自动记录模式
- 后续对话不再自动写入正文
- 不自动修改日志文件，除非用户另外要求更新

## 会话状态规则

记录模式属于**当前会话状态**，不等于文件状态。

- `$session-log 初始化日志`：开启当前线程的记录模式
- `$session-log 关闭记录`：关闭当前线程的记录模式
- 如果线程上下文重置或丢失，需要用户再次显式调用 `$session-log` 以重新开启记录

## 文件格式

应保持日志文件结构如下，以便后续稳定更新：

```md
# Session Log

<!-- SESSION_OVERVIEW_START -->
## 会话概览
- 创建时间：
- 最近更新时间：
- 当前项目目录：
- 当前状态：
- 当前阶段：
- 累计记录数：
<!-- SESSION_OVERVIEW_END -->

---

<!-- SESSION_ENTRIES_START -->
## YYYY-MM-DD HH:mm:ss

- 任务目标：
- 我让你做的事情：
- 你遇到的问题：
- 我给出的解决方式：
- 你的反馈：
- 实际最终解决方式：
- 当前状态：
- 遗留问题 / 下一步：
- 关键命令 / 报错 / 文件：
<!-- SESSION_ENTRIES_END -->
```

规则：

- 新的正文记录始终追加到 `<!-- SESSION_ENTRIES_END -->` 之前
- 不覆盖已有正文历史记录
- 除文件标题 `# Session Log` 外，其余字段名与说明文字默认使用简体中文
- 文件应使用 UTF-8 编码保存，确保中文正常显示

## 正文写入规则

当追加一条任务记录时：

1. 使用**摘要式记录**，不要复制整段完整对话
2. 默认使用简体中文
3. 能填写时优先填写以下字段：
   - 任务目标
   - 我让你做的事情
   - 你遇到的问题
   - 我给出的解决方式
   - 你的反馈
   - 实际最终解决方式
   - 当前状态
   - 遗留问题 / 下一步
   - 关键命令 / 报错 / 文件
4. 若部分字段没有内容，可写 `无`，或保持简短，不要编造信息
5. 不记录密码、token、cookie、密钥等敏感信息；如确需提及，只保留脱敏摘要

## 概览更新规则

概览区采用**手动更新优先**原则，不要求每次普通对话都刷新。

- 仅在用户明确要求更新概览时更新
- 概览应反映整体任务状态，而不是单轮对话全文
- 推荐状态值：
  - `进行中`
  - `已完成`
  - `阻塞中`
  - `部分完成`
- 当前阶段建议使用简短中文短语，例如：
  - `需求确认`
  - `实现中`
  - `测试中`
  - `已收尾`
- 如有条件，应根据正文实际条目重新计算“累计记录数”

## 推荐工具

优先使用本 skill 自带的 helper 脚本，以更稳定地完成初始化、状态检查和概览更新：

- `scripts/session_log.py init`
- `scripts/session_log.py status`
- `scripts/session_log.py update-overview`

如果在 Windows shell 中传递中文参数可能带来编码风险，可优先使用英文别名参数：

- `--status in-progress|completed|blocked|partial`
- `--phase-code init|requirements|implementing|testing|wrap-up`

如果不使用脚本而直接修改 Markdown，也必须保留标记块不变：

- `<!-- SESSION_OVERVIEW_START -->`
- `<!-- SESSION_OVERVIEW_END -->`
- `<!-- SESSION_ENTRIES_START -->`
- `<!-- SESSION_ENTRIES_END -->`
